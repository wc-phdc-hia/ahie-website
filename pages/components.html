Components

The architectural design of the AHIE project incorporates several key components which facilitate the exchange of Health Information within a replicable and standardised framework.

These components include interoperability technologies, data harmonisation strategies and patient-level healthcare management applications, which were identified as the cornerstones for successful Health Information Exchange.

Together these components form a structure which allows both flexibility of communication between disparate systems, as well as scalability on introduction of new applications to the health information exchange implementation.

Inclusion of components were guided by the following principles:

Open Source Technology

The set of technologies that would be developed and matured as part of this project would be open source and easily shareable, enabling a meaningful contribution to global technology projects. Where possible existing technologies that were already operational at a national level in South Africa (as well as several African countries) were leveraged.

Standards-Based

The implementation of the resulting set of technologies or technology stack should be easily replicable, with a standards-based approach allowing interoperability between systems and providing a foundation for scaled implementations.

Context-Appropriate

This set of technologies would be context-appropriate, taking into consideration that use of electronic medical records at the point of care might not be pervasive, that shared health records might need to be derived from various available data sources, and that the solution architecture should be flexible enough to allow for low resource settings where dependence on infrastructure is minimised.

Off-Line Capability

The proposed technology stack would allow off-line capability while still providing the advantages of data exchange, e.g. demographic and laboratory data, when and where possible.

Person-Level Data Harmonisation

Data harmonization would be done at person-level, enabling cascade reporting for individuals throughout the journey of disease management and care. The scale up of a unique national patient identifier was key to the success of the project, allowing the linkage of data from multiple sources in the HIE environment.

Data Integrity

There would be a rigorous focus on data stewardship.

Data governance of patient data would remain the responsibility of the National and Provincial Departments of Health and no patient data (identified, anonymised or aggregated) would be made available by this project.
